# QuantClaw 开发计划

> QuantClaw v0.2.0 → v1.0.0 路线图
> 基于 OpenClaw 参考实现分析

---

## 当前状态 (v0.2.0)

**已实现功能**（324 个测试通过）：
- 核心：AgentLoop、MemoryManager、SkillLoader、PromptBuilder、Config、SignalHandler
- LLM：OpenAI + Anthropic 提供商，支持模型路由
- 网关：WebSocket RPC（11 个方法）+ HTTP REST API（10 个端点）
- 频道：Discord + Telegram（Node.js 适配器）
- 会话：JSONL 持久化，完整对话历史
- 工具：文件读写/编辑、Shell 执行、工具链、权限、安全沙箱
- MCP：服务端 + 客户端 + 工具管理器
- CLI：gateway/agent/sessions/status/health/config/skills/doctor 命令

---

## 阶段 1 — 基础加固 (v0.3.0)

> 目标：强化核心架构，提升稳定性和开发体验

### 1.1 多 Agent 配置文件
- 每个 Agent 独立配置目录（`~/.quantclaw/agents/{name}/`）
- Agent 专属 SOUL.md、技能、工具权限
- 通过 CLI 标志选择 Agent（`--agent`）
- 默认 Agent 回退机制

### 1.2 认证配置与模型故障转移
- 每个提供商支持多个 API 密钥配置文件
- 在 429/500 错误时自动故障转移
- 失败认证配置的冷却计时器
- 模型回退链（例如 claude-opus → claude-sonnet → gpt-4o）

### 1.3 配置模式验证
- 加载配置时进行严格模式验证（nlohmann/json_schema）
- 无效配置的详细错误消息
- 增强 `doctor` 命令：验证配置、检查连接性、验证依赖
- 支持破坏性更改的配置迁移

### 1.4 入门向导
- 交互式 `quantclaw onboard` 命令
- 分步引导：API 密钥设置 → 模型选择 → 频道配置 → 守护进程安装
- 自动生成配置文件
- 连接性验证

### 1.5 日志与诊断
- 基于子系统的日志级别（gateway=info，agent=debug 等）
- 结构化 JSON 日志输出选项
- `--verbose` / `--quiet` CLI 标志
- 日志轮转支持

**预计新增测试**：约 80 个

---

## 阶段 2 — 插件系统 (v0.4.0)

> 目标：使 QuantClaw 可扩展，无需修改核心代码

### 2.1 插件架构
- 插件清单格式（JSON），包含元数据、能力、依赖
- 插件注册表，生命周期管理（load → init → start → stop → unload）
- 插件插槽：频道、技能、工具、钩子
- 命名空间隔离（`plugin__{name}__{resource}`）

### 2.2 插件 SDK（C API）
- 插件稳定 C ABI（共享库 .so/.dylib/.dll）
- 插件 SDK 头文件，带版本 API
- 回调注册：`qc_register_channel()`、`qc_register_tool()` 等
- 示例插件模板

### 2.3 钩子系统
- 事件钩子：`before_agent_start`、`after_tool_call`、`before_reply`、`on_error`
- 钩子优先级排序
- 异步钩子执行支持
- 内置钩子用于日志和指标

### 2.4 技能插件加载
- 从插件目录加载技能
- 技能元数据，环境门控
- 技能插件热重载支持

**预计新增测试**：约 60 个

## 阶段 3 — 频道扩展 (v0.5.0)

> 目标：支持 Discord/Telegram 以外的主流消息平台

### 3.1 频道插件框架
- 标准化 C++ 频道接口，虚方法
- 频道生命周期管理（连接、重连、断开）
- 消息标准化层（特定平台 → 统一格式）
- 每个频道的速率限制
- 每个频道的白名单/黑名单

### 3.2 Slack 频道
- Slack Bolt 兼容适配器
- 斜杠命令支持
- 线程感知对话
- 富文本消息格式（blocks）

### 3.3 WhatsApp 频道
- WhatsApp Business API / Baileys 适配器（Node.js）
- 媒体消息支持（图片、音频、文档）
- 群聊处理

### 3.4 Web 频道
- 基于 HTTP 的 Web 聊天端点
- SSE（Server-Sent Events）用于流式响应
- 可嵌入小部件支持
- 会话令牌管理

### 3.5 Matrix 频道
- Matrix Client-Server API 适配器
- 端到端加密支持（libolm）
- 基于房间的对话

### 3.6 Signal / iMessage（可选，依赖平台）
- Signal 协议适配器
- iMessage（仅 macOS，通过 AppleScript 桥接）

**预计新增测试**：约 100 个

## 阶段 4 — 记忆与智能 (v0.6.0)

> 目标：长期记忆和上下文感知响应

### 4.1 基于嵌入的记忆
- 向量存储抽象层
- SQLite + vec 扩展用于本地嵌入
- 记忆索引和搜索（`memory search <query>`）
- 自动对话摘要

### 4.2 记忆插件
- 可插拔的记忆后端（SQLite-vec、LanceDB、FAISS）
- 记忆导入/导出
- 记忆修剪和相关性衰减

### 4.3 增强会话上下文
- 跨会话上下文检索
- 用户偏好学习
- 对话主题跟踪
- 会话摘要生成

### 4.4 链接理解
- URL 元数据提取（标题、描述、OG 标签）
- HTML 内容解析（Readability 风格）
- 自动链接预览到 Agent 上下文

**预计新增测试**：约 70 个

## 阶段 5 — 媒体与富内容 (v0.7.0)

> 目标：处理图片、音频、文档和富媒体

### 5.1 媒体管道
- 媒体类型检测（MIME）
- 图像处理（调整大小、缩略图、格式转换）通过 stb_image/libpng
- 音频处理（格式转换、重采样）通过 libsndfile
- 所有频道的文件附件处理

### 5.2 媒体理解
- 通过多模态 LLM 进行图像分析（视觉 API）
- 音频转录（Whisper API 集成）
- 文档解析（PDF、DOCX 文本提取）

### 5.3 TTS（文本转语音）
- TTS 提供商抽象
- OpenAI TTS API 集成
- 音频响应传递到频道

### 5.4 图像生成
- 图像生成工具（DALL-E / Stable Diffusion API）
- 图像传递到频道

**预计新增测试**：约 50 个

## 优先级与依赖关系

```
阶段 1（基础）──┬── 阶段 2（插件）──── 阶段 3（频道）
               │                          │
               ├── 阶段 4（记忆）──── 阶段 6（高级功能）
               │                          │
               └── 阶段 5（媒体）──── 阶段 7（Web UI）
                                          │
                                   阶段 8（生产）
```

- 阶段 1 是所有后续阶段的前置条件
- 阶段 2-5 可以在阶段 1 完成后并行开发
- 阶段 6-7 依赖于早期阶段
- 阶段 8 是最终稳定化阶段

---

## 原则

1. **不使用 Boost** — 保持依赖轻量化
2. **仅 C++17** — 不使用 C++20 特性以获得最大编译器兼容性
3. **测试优先** — 每个功能都配备测试
4. **OpenClaw 兼容** — 尽可能保持协议兼容性
5. **单二进制目标** — 最小化运行时依赖，便于部署
6. **增量交付** — 每个阶段都是独立可用的
